## üèóÔ∏è SOLID Principles

### S - Single Responsibility Principle
- Each class should have only one reason to change
- Examples: User management, Email service separation

### O - Open/Closed Principle  
- Classes should be open for extension, closed for modification
- Examples: Shape drawing, Discount calculation

### L - Liskov Substitution Principle
- Derived classes must be substitutable for base classes
- Examples: Bird hierarchy, Vehicle inheritance

### I - Interface Segregation Principle
- Clients shouldn't depend on interfaces they don't use
- Examples: Printer interfaces, Worker roles

### D - Dependency Inversion Principle
- Depend on abstractions, not concretions
- Examples: Database abstraction, Service injection
